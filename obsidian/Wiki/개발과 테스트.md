# 개발과 테스트

## 개발 명령

```bash
npm start
npm test
```

`package.json` 기준으로 `start`와 `dev`는 모두 `node server.js`를 실행합니다. 테스트는 Node.js 내장 test runner를 사용합니다.

## 테스트 구성

`tests/runasis.test.js`는 단순 unit test뿐 아니라 서버 API, 로컬 저장소 처리, UI 렌더링 함수 일부를 함께 검증합니다. 2026-06-08 triage 기록 기준으로 `npm test` 77개가 통과했습니다.

대표 커버리지입니다.

- pace, distance, date formatting helper
- dashboard range 계산
- Strava fetch 요청 shape
- static file path escape 방어
- JSON body 제한과 content type 확인
- host/origin/CSRF 방어
- detail/raw detail/raw stream 저장 상태
- personal best, time best, pace best 계산
- excluded record 저장과 표시
- dashboard, activity list, personal best, Riegel UI 동작 일부

## 유지보수 리스크

- `server.js`와 `public/app.js`가 큰 파일이라 변경 전 관련 테스트 범위를 먼저 찾는 것이 좋습니다.
- Strava 설정, 토큰, 로컬 데이터 삭제는 사용자 데이터에 직접 영향을 줍니다.
- personal best와 Riegel projection은 계산 로직과 UI 차트가 함께 엮여 있습니다.
- SVG chart label, linear/log scale, 반응형 layout은 작은 변경에도 회귀가 생길 수 있습니다.
