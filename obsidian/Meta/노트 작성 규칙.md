# 노트 작성 규칙

## 기본 원칙

- 모든 노트는 한국어로 작성합니다.
- 내부 이동은 Obsidian wiki link를 사용합니다.
- 코드 파일 경로는 일반 텍스트로 표기합니다.
- `.obsidian/` 설정 파일과 `환영합니다!.md`는 사용자가 직접 요청하지 않는 한 수정하지 않습니다.
- `.env`와 `data/` 내부 내용은 읽거나 노트에 복사하지 않습니다.

## 폴더 역할

- `Wiki/`: 개념 설명, 아키텍처, 기능별 해설
- `Todo/`: 사용자가 직접 입력한 Todo와 목차
- `Triage/`: 현재 triage 요약, 날짜별 실행 기록, triage가 발견한 제안
- `Meta/`: 노트 작성 규칙과 템플릿

## frontmatter 템플릿

Wiki 노트:

```yaml
---
type: wiki
tags:
  - runasis
updated: YYYY-MM-DD
---
```

MOC 노트:

```yaml
---
type: moc
tags:
  - runasis
updated: YYYY-MM-DD
---
```

사용자 Todo 카드:

```yaml
---
type: todo
status: candidate
priority: medium
source: docs/loop/state.md
updated: YYYY-MM-DD
---
```

Triage 제안:

```yaml
---
type: triage-suggestion
status: candidate
priority: medium
source: docs/loop/state.md
updated: YYYY-MM-DD
---
```

Triage 기록:

```yaml
---
type: triage
date: YYYY-MM-DD
source: docs/loop/state.md
updated: YYYY-MM-DD
---
```

## Todo 카드 작성 규칙

사용자가 직접 입력한 Todo 하나는 노트 하나로 만듭니다. triage가 발견한 제안은 Todo에 넣지 않고 `Triage/Suggestions/` 아래에 둡니다. 최소 섹션은 다음 순서를 따릅니다.

- 배경
- 관련 파일
- 완료 기준
- 검증
- 관련 노트

상태 값은 `candidate`, `active`, `done`, `deferred` 중 하나를 사용합니다.

## Triage 기록 규칙

- 최신 요약은 `Triage/현재 triage 상태.md`에 유지합니다.
- 날짜별 실행 기록은 `Triage/History/YYYY-MM-DD 설명.md`에 추가합니다.
- triage가 발견한 제안은 `Triage/Suggestions/` 아래 카드로 관리합니다.
- 원본은 `docs/loop/state.md`로 보고, Obsidian 노트는 탐색과 연결을 위한 요약으로 유지합니다.

## 민감정보 규칙

- Strava 인증 정보, 토큰, 개인 활동 원본 JSON은 노트에 넣지 않습니다.
- 민감 파일은 경로와 역할만 기록합니다.
- 예시가 필요하면 실제 값이 아닌 구조 설명으로만 작성합니다.
